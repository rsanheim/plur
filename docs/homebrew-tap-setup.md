# Homebrew Tap Setup Guide

This guide explains how to set up the Homebrew tap repository for plur distribution.

## Repository Setup

### 1. Create the Tap Repository

Create a new GitHub repository named `homebrew-tap`:
* Repository name: `rsanheim/homebrew-tap`
* Description: "Homebrew formulas for plur and other tools"
* Visibility: Public
* Initialize with README: Yes

### 2. Repository Structure

The tap repository should have this structure:
```
homebrew-tap/
├── Formula/          # Directory for Homebrew formulas
│   └── plur.rb      # Will be auto-generated by GoReleaser
├── README.md        # Usage instructions for the tap
└── .github/         # Optional: GitHub Actions for testing
```

### 3. Create Initial README

Create a `README.md` in the tap repository:

```markdown
# Homebrew Tap for rsanheim

This tap contains Homebrew formulas for my tools.

## Installation

Add this tap:
```bash
brew tap rsanheim/tap
```

## Available Formulas

### plur
Fast parallel test runner for Ruby/RSpec.

```bash
brew install plur
```

Or install directly:
```bash
brew install rsanheim/tap/plur
```

## Updating

```bash
brew update
brew upgrade plur
```
```

### 4. GitHub Token Setup

For GoReleaser to push to the tap repository:

1. Create a Personal Access Token:
    * Go to GitHub Settings → Developer settings → Personal access tokens → Tokens (classic)
    * Click "Generate new token (classic)"
    * Name: `TAP_GITHUB_TOKEN`
    * Scopes required:
        * `repo` (full control of private repositories)
        * `workflow` (if the tap repo has GitHub Actions)
    * Generate and copy the token

2. Add to plur repository secrets:
    * Go to plur repository → Settings → Secrets and variables → Actions
    * Click "New repository secret"
    * Name: `TAP_GITHUB_TOKEN`
    * Value: Paste the token
    * Click "Add secret"

## Testing the Setup

### Manual Test Release

1. Create a test tag:
```bash
git tag v0.1.0-test
git push origin v0.1.0-test
```

2. Run GoReleaser locally (dry run):
```bash
goreleaser release --snapshot --clean --skip=publish
```

3. Run GoReleaser with publishing (requires token):
```bash
export TAP_GITHUB_TOKEN=your_token_here
goreleaser release --clean
```

### Verify Formula Generation

After a successful release, check the tap repository:
* Formula should appear in `Formula/plur.rb`
* Formula should contain:
  * Correct version
  * SHA256 checksums
  * Download URLs
  * Installation instructions

### Test Installation

```bash
# Remove any existing installation
brew uninstall plur 2>/dev/null || true

# Add your tap
brew tap rsanheim/tap

# Install plur
brew install plur

# Verify
plur --version
```

## Troubleshooting

### Token Permission Issues
* Error: "Resource not accessible by integration"
* Solution: Ensure token has `repo` scope and is correctly set as `TAP_GITHUB_TOKEN`

### Formula Not Updating
* Check GoReleaser logs for errors
* Verify the tap repository exists and is public
* Ensure the token has write access to the tap repository

### Installation Fails
* Run `brew update` first
* Check formula syntax: `brew audit rsanheim/tap/plur`
* View formula: `brew cat rsanheim/tap/plur`

## Optional: Scoop Bucket Setup

Similar process for Windows Scoop:

1. Create `rsanheim/scoop-bucket` repository
2. Add `SCOOP_GITHUB_TOKEN` secret
3. GoReleaser will auto-generate manifests

## Maintenance

### Formula Updates
* GoReleaser automatically updates the formula on each release
* No manual intervention needed for version bumps

### Adding Dependencies
Update `.goreleaser.yml` if plur needs runtime dependencies:
```yaml
brews:
  - dependencies:
      - name: ruby
        type: optional
```

### Versioned Formulas
GoReleaser can maintain versioned formulas:
* `plur.rb` - Always latest
* `plur@1.0.rb` - Specific version (if configured)
